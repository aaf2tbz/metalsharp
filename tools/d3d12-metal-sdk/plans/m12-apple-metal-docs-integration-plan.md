# M12 Apple Metal Documentation Integration Plan

Date: 2026-06-16
Branch: `fix/m12-shader-probe-lab`
Phase: 3.5

## Purpose

Phase 3.5 converts Apple's official Metal API guidance into concrete M12 diagnostics and audit gates before broad Phase 4 D3D12/DXGI probe expansion.

This is not a gameplay-launch phase. It is a diagnostic hardening phase focused on Elden Ring, Subnautica 2, and Armored Core VI evidence already collected in M12 caches and run artifacts.

## Official Apple API anchors

Primary documentation hubs:

- `https://developer.apple.com/documentation/metal`
- `https://developer.apple.com/documentation/metalfx`
- `https://developer.apple.com/documentation/xcode`

M12 workstreams map to these official Metal areas:

| M12 area | Apple API / tool area | Why it matters |
|---|---|---|
| Shader source compile failures | `MTLDevice`, `MTLLibrary`, `MTLCompileOptions`, `NSError` | AC6 currently has four MSL compile failures with `MTLLibraryErrorDomain Code=3`; diagnostics must preserve full Apple error context. |
| Render/compute PSO creation | `MTLRenderPipelineDescriptor`, `MTLComputePipelineDescriptor`, `MTLPipelineOption`, pipeline reflection | Elden Ring render PSO correctness now depends on reflected stage-in vertex descriptors and descriptor-affecting cache keys. |
| Vertex descriptors | `MTLVertexDescriptor`, `MTLVertexAttributeDescriptor`, `MTLVertexBufferLayoutDescriptor`, `[[stage_in]]`, `[[attribute(n)]]` | Missing/wrong vertex attributes previously broke Elden Ring render PSOs. |
| Command-buffer hangs/errors | `MTLCommandBuffer`, `MTLCommandBuffer.error`, `MTLCommandBufferError`, `MTLCommandBufferEncoderInfo` | Elden Ring character creation is a live hang, not an exit/crash; Schedule-like OOM/page-fault cases must not collapse into generic failure. |
| Resource hazards | `MTLResource`, `MTLHeap`, `MTLHazardTrackingMode`, `useResource(s)`, `useHeap(s)`, fences/events/shared events | Phase 4 probes must target binding/heap/barrier/synchronization gaps, not generic API smoke only. |
| Cache/archive freshness | `MTLBinaryArchive`, `MTLBinaryArchiveDescriptor`, pipeline descriptors | Force-DXIL-source-compile and reflected descriptors must not silently reuse stale Metal artifacts. |
| Capture/validation | Metal API Validation, GPU Capture, Metal System Trace/Instruments | Validation runs must record which Apple debug tooling was active. |
| MetalFX | MetalFX descriptors/support checks | MetalFX must be off or explicitly capability-gated for visual correctness validation. |

## Phase 4 probe requirements generated by Apple-doc gate

Before broad Phase 4 expansion, probes must include these Apple-doc-backed areas:

1. **Command-buffer diagnostics probe**
   - force a known success and known failure path,
   - record command-buffer status, `NSError.domain`, numeric code, localized description, and encoder info when present.

2. **Resource declaration/hazard probe**
   - cover indirect resources accessed through argument/descriptor tables,
   - verify required resource/heap usage declarations are represented in M12 code paths,
   - exercise aliasing/placed-resource reuse across command-buffer boundaries.

3. **Heap/storage synchronization probe**
   - upload/readback/managed-resource sequence with explicit fence/event ordering evidence.

4. **Vertex descriptor reconstruction probe**
   - build a PSO with mixed slots, offsets, per-instance step rates, and sparse attributes,
   - compare D3D12 input layout, generated MSL attributes, reflected Metal attributes, and final `MTLVertexDescriptor`.

5. **Binary archive/cache freshness probe**
   - same shader with descriptor-affecting state changes must not reuse incompatible artifacts,
   - changed generated MSL must invalidate stale metallib/error artifacts.

6. **Capture/validation record probe**
   - every bounded run artifact records whether Metal API Validation, GPU capture, or Metal System Trace was requested.

## Stop rules

- Do not mutate live Elden Ring, AC6, or Subnautica shader caches from offline tools.
- Do not reopen DX11/PEAK/Schedule as active targets in this phase.
- Do not treat drawn/present counts as visual correctness.
- Do not start broad Phase 4 probes until this plan's output has been reviewed and any missing critical diagnostics are either implemented or recorded as deferred risks.
